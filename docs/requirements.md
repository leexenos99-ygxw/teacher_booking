# 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目背景
当前家校沟通教育场景存在显著痛点：
- **家长侧**：希望随时联系老师、及时了解孩子在校学习情况
- **教师侧**：长期被碎片化消息打扰，工作时间与私人生活时间混淆，家校沟通效率低下、无统一管理渠道

### 1.2 项目目标
构建教师与家长之间清晰、健康的沟通边界，建立规范化家校沟通机制。

### 1.3 核心理念
摒弃即时闲聊式沟通，建立规范化沟通流程：
**预约 → 确认 → 限时沟通 → 全程记录 → 事后反馈**

## 4. 非功能需求

### 4.1 性能需求
- 页面响应时间 < 2秒
- API响应时间 < 500ms
- 支持1000+并发用户
- WebSocket消息延迟 < 100ms

### 4.2 安全需求
- 密码加密存储（BCrypt）
- JWT令牌认证
- RBAC权限控制
- SQL注入防护
- XSS跨站脚本防护
- 接口限流防刷

### 4.3 可用性需求
- 系统可用性 99.5%+
- 全站响应式设计（PC/平板/手机）
- 完善的错误提示
- 操作日志可追溯

### 4.4 可维护性需求
- 代码分层清晰
- 统一接口返回格式
- 全局异常处理
- 完善的日志记录
- Swagger接口文档

## 5. 核心业务流程

### 5.1 预约流程
```
家长发起预约 → 教师待确认 → 教师接受/拒绝 → 预约成功/失败
                                    ↓
                              预约时间到达 → 开始沟通 → 结束沟通 → 评价归档
```

### 5.2 注册流程
```
用户选择角色 → 填写注册信息 → 提交注册 → 自动分配角色权限
                                      ↓
                              教师 → 待管理员审核 → 审核通过/驳回
```

### 5.3 沟通流程
```
预约时间到达 → 双方进入沟通室 → 实时消息收发 → 限时倒计时 → 沟通结束 → 生成记录
```
